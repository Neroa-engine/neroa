# Strategy Extraction Framework v1

This namespace adds the hidden strategy-phase architecture that sits above Neroa's extraction, branching, question, and readiness contracts.

What it does:

- defines the primary strategy spine across identity, naming, idea, product definition, function, user, goals, surfaces, and systems/constraints
- defines explicit naming, ideation, narrowing, and safety branches
- defines optional overlays for brandability, domain intent, market/crowding, monetization, compliance/security, AI, data, mobile/device, and admin/ops
- provides a canonical strategy-framework output that now travels with the hidden intelligence bundle
- provides summary-safe derived outputs for future workspace rendering
- provides a minimum workspace-handoff coverage model grounded in structured truth

What it does not do:

- it does not change visible `/start` wording, sequencing, or layout
- it does not replace live question selection
- it does not populate workspace/project summaries from raw transcript replay
- it does not implement browser/runtime or Command Center behavior

How future passes should use it:

1. use these phase and branch definitions as the canonical hidden strategy-discovery model
2. compare shadow traces and visible planning behavior against this framework before changing live strategist behavior
3. derive workspace-safe summary fields from these structured outputs, not copied chat fragments
